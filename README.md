## ✦ PEMBUKA

---

**[Masuk ke panggung. Berdiri tegak, senyum, buat kontak mata dengan audiens sebelum mulai berbicara.]**

Halo semuanya — selamat datang.

Sebelum saya mulai, saya ingin kalian bayangkan sebuah skenario sebentar.

Bayangkan seorang wanita // sedang hamil 7 bulan. // Setiap pagi, ia bangun lebih awal, mencatat tekanan darahnya, menginput berat badannya, mengingat jadwal kontrol dokter minggu ini, dan memantau gerakan bayinya satu per satu. // Sendirian.

Sementara itu, suaminya — yang juga sangat peduli — // hanya bisa bertanya satu kalimat di malam hari: // *"Hari ini gimana, sayang?"*

Bukan karena ia tidak mau terlibat. // Tapi karena **tidak ada jembatan** yang menghubungkan mereka berdua // dalam perjalanan yang harusnya ditempuh bersama ini.

**Inilah masalah yang kami angkat hari ini.**

Dan inilah mengapa kami membangun **CARE** — *Couple Assisted Reproductive Experience.* //

Sebuah aplikasi pregnancy tracker // yang tidak hanya menjadi asisten bagi ibu hamil, // tetapi juga **mengajak suami masuk ke dalam perjalanan itu**.

## ✦ BAGIAN 1 — LATAR BELAKANG

### *Kenapa ini penting? Apa yang sedang terjadi di luar sana?*

---

**[Nada serius, informatif. Bergerak sedikit ke sisi panggung agar terasa lebih dinamis.]**

Mari kita bicara tentang data.

Pasar aplikasi pregnancy tracker global // saat ini bernilai lebih dari **USD 300 juta** — // dan diproyeksikan akan tumbuh hampir **5 kali lipat** menjadi USD 1,38 miliar pada tahun 2033. // Angka pertumbuhannya? // **18,4% per tahun.** //

Ini bukan angka kecil. //

Ini adalah sinyal yang sangat jelas // bahwa dunia sedang bergerak // ke arah **digitalisasi kesehatan maternal**. // Ibu-ibu hamil di seluruh dunia — termasuk di Indonesia — // semakin sadar, semakin melek teknologi, // dan semakin aktif mencari bantuan digital // untuk menemani perjalanan kehamilan mereka.

Tren ini juga didorong oleh beberapa faktor besar:

**Pertama** — penetrasi smartphone yang terus meningkat. // Di India saja, jumlah pengguna smartphone // melonjak dari 500 juta di 2022 // menjadi lebih dari 700 juta di 2024. // Dan ini berbanding lurus dengan peningkatan unduhan aplikasi kehamilan sebesar **156%** // hanya dalam dua tahun.

**Kedua** — meningkatnya kesadaran masyarakat // tentang pentingnya *preventive prenatal care.* // Ibu hamil tidak lagi menunggu ada masalah // baru pergi ke dokter. // Mereka ingin **memantau secara aktif**, // secara *real-time*, // di genggaman tangan mereka sendiri.

**Ketiga** — integrasi telehealth. // Semakin banyak aplikasi yang mulai menggabungkan fitur // konsultasi virtual dengan tenaga medis langsung dari aplikasi.

Tapi di balik semua tren positif ini — // ada satu **celah besar** yang belum banyak diisi.

**Keterlibatan pasangan.**

Studi-studi terkini secara konsisten menunjukkan // bahwa **dukungan dari suami atau pasangan** // adalah salah satu faktor paling signifikan // dalam menentukan kualitas kesehatan ibu selama masa kehamilan. //

Ketika pasangan hadir, aktif, dan terlibat — // tingkat stres ibu menurun, // kepatuhan pada jadwal prenatal meningkat, // dan secara umum, pengalaman kehamilan menjadi // jauh lebih positif dan sehat. //

Namun kenyataannya? // Hampir **tidak ada** platform digital // yang secara khusus dirancang // untuk memfasilitasi kolaborasi ini. //

Aplikasi yang ada hari ini // hampir semuanya berbicara **kepada ibu**, // bukan **kepada pasangan suami-istri sebagai satu unit**. //

**Dan di sanalah CARE hadir.**

## ✦ BAGIAN 2 — MENGAPA KAMI MEMILIH PROJEK INI

### *Ini bukan sekadar pilihan teknis. Ini adalah pilihan yang bermakna.*

---

**[Berdiri di tengah panggung. Berbicara lebih personal, lebih dalam.]**

Sekarang, pertanyaan yang wajar untuk ditanyakan adalah: // *"Dari sekian banyak ide projek — // kenapa ini yang kalian pilih?"*

Jawabannya sederhana tapi kuat.

**Karena ini nyata.**

Kehamilan bukan sebuah kondisi medis semata. // Ia adalah sebuah **perjalanan hidup** // yang membawa perubahan fisik, emosional, dan psikologis // pada dua orang sekaligus — ibu **dan** ayah. //

Namun selama ini, // dukungan digital hanya memperhatikan **satu sisi** dari persamaan itu. //

Kami melihat tiga alasan konkret // mengapa projek ini layak dan perlu:

**Alasan pertama — ada masalah nyata yang belum terpecahkan.**
Ibu hamil membutuhkan sistem pencatatan yang terstruktur. // Suami membutuhkan cara untuk *stay informed* // tanpa harus terus-menerus bertanya. // Dan keduanya membutuhkan **satu platform** // yang bisa menyatukan informasi itu // secara elegan dan mudah.

**Alasan kedua — ada permintaan pasar yang besar dan belum terlayani.**
Seperti yang kita lihat tadi, // pasar aplikasi kehamilan sedang tumbuh sangat pesat. // Dan segmen "couple-centered" atau "family-centered" // dalam ekosistem ini // hampir belum ada yang menggarap secara serius. // Ini adalah *blue ocean* yang nyata.

**Alasan ketiga — ada dampak kesehatan yang bisa diukur.**
Ini bukan sekadar fitur yang bagus secara teknis. // Jika seorang suami bisa melihat // grafik perkembangan kandungan istrinya, // menerima notifikasi jadwal kontrol dokter, // dan berpartisipasi dalam pencatatan harian — // itu bukan sekadar fitur. // Itu adalah **investasi pada kesehatan keluarga.**

Inilah mengapa kami tidak memilih projek yang lebih sederhana. // Kami memilih yang bermakna.

## ✦ BAGIAN 3 — REFERENSI DAN POSITIONING

### *Di mana posisi CARE dalam ekosistem yang sudah ada?*

---

**[Bergerak sedikit, nada analitis dan percaya diri.]**

Tentu saja, kami tidak membangun ini di ruang hampa. // Kami belajar dari apa yang sudah ada — // dan kemudian kami **memilih untuk berbeda**.

Mari kita lihat lanskap yang sudah ada sekarang.

**Di kategori health tracking umum**, // kita mengenal aplikasi seperti *MyFitnessPal*, *Apple Health*, atau *Google Fit.* // Mereka sangat baik dalam mencatat // kalori, aktivitas fisik, dan metrik tubuh secara umum. // Tapi mereka dirancang untuk **individu**, // bukan untuk **pasangan**. // Dan mereka tidak punya konteks khusus untuk kehamilan.

**Di kategori habit tracker**, // ada *Habitica*, *Streaks*, atau *Done.* // Mereka efektif membangun kebiasaan — // tapi lagi-lagi, // sifatnya personal dan generik. // Tidak ada konten prenatal, // tidak ada konteks medis, // dan tidak ada fitur berbagi dengan pasangan.

**Di kategori pregnancy tracker sendiri**, // ada nama-nama besar seperti // *What to Expect*, *The Bump*, *BabyCenter*, dan *Ovia Health.* // Ini adalah kompetitor paling langsung kami. // Dan mereka melakukan banyak hal dengan sangat baik — // konten edukasi, pemantauan perkembangan janin per minggu, // dan artikel kesehatan yang informatif.

**Tapi — dan ini adalah "tapi" yang besar —** // hampir semua dari mereka // **hanya berbicara kepada ibu.** //

Fitur untuk pasangan, kalau ada, // sifatnya sangat terbatas. // Suami bisa melihat, tapi tidak bisa berkontribusi. // Suami diberi konten, tapi tidak diberi peran. //

**Di sinilah CARE hadir dengan positioning yang jelas:**

CARE bukan sekadar pregnancy tracker. // CARE adalah **platform kolaborasi kehamilan untuk pasangan suami-istri.** //

Kami bukan menggantikan What to Expect atau The Bump. // Kami mengisi ruang yang mereka tinggalkan — // yaitu **keterlibatan aktif pasangan** // dalam satu ekosistem yang terintegrasi dan mudah digunakan. //

Dalam peta kompetisi, // kalau aplikasi lain berbicara kepada *ibu hamil*, // CARE berbicara kepada ***keluarga yang sedang menunggu kelahiran*.**

## ✦ BAGIAN 4 — IMPLEMENTASI NYATA CARE

### *Ini bukan hanya konsep. Ini sudah ada.*

---

**[Nada antusias. Ini adalah bagian paling konkret — tunjukkan keyakinan.]**

Sekarang saya ingin berbicara tentang apa yang sudah **benar-benar kami bangun.**

Karena CARE bukan sekadar ide di atas kertas. // CARE sudah ada sebagai aplikasi yang berjalan — // dengan frontend berbasis **React** // dan backend berbasis **Django REST API.**

---

### 🎯 Target Market

Target utama kami adalah pasangan suami-istri // di rentang usia **22–35 tahun**, // yang sedang menjalani masa kehamilan pertama atau kedua, // yang melek teknologi, // dan yang aktif menggunakan smartphone sebagai bagian dari kehidupan sehari-hari.

Secara geografis, kami berfokus pada **pasar Indonesia** terlebih dahulu, // dengan mempertimbangkan penetrasi smartphone yang terus naik // dan belum adanya pemain lokal yang kuat // di segmen ini.

---

### 🔧 Fitur MVP yang Sudah Ada Sekarang

Berikut adalah fitur-fitur inti yang sudah berjalan pada versi saat ini:

**Pertama — Sistem Autentikasi Berbasis Pasangan.**
Ibu dan suami mendaftar secara terpisah, // lalu terhubung melalui kode unik partner. // Begitu terhubung, keduanya berada dalam satu ekosistem yang saling terbagi.

**Kedua — Health Log Harian.**
Ibu bisa mencatat kondisi hariannya — // tekanan darah, berat badan, keluhan, mood, dan catatan umum. // Semua tersimpan dan bisa diakses secara historis // dalam bentuk grafik yang mudah dibaca.

**Ketiga — Appointment Tracker.**
Jadwal kontrol ke dokter atau bidan // bisa dicatat, diedit, dan dipantau // oleh keduanya — ibu maupun suami. // Tidak ada lagi jadwal yang terlewat karena lupa.

**Keempat — Partner Dashboard.**
Suami punya tampilan tersendiri // yang menampilkan ringkasan kondisi istrinya. // Bukan sekadar melihat data mentah — // tapi mendapatkan konteks yang bermakna // tentang bagaimana kondisi istri hari ini.

**Kelima — Visualisasi Perkembangan Kehamilan.**
Tersedia tampilan progress kehamilan per minggu // yang menunjukkan milestone perkembangan janin // secara visual dan informatif.

**Keenam — Demo Mode.**
Kami juga membangun fitur demo // di mana siapapun bisa mencoba aplikasi ini // tanpa perlu mendaftar terlebih dahulu, // menggunakan akun demo yang sudah disiapkan.

---

### 🚀 Fitur yang Direncanakan ke Depan

Dan ini belum selesai. // Roadmap kami ke depan mencakup beberapa fitur yang kami yakini // akan semakin memperkuat proposisi nilai CARE:

**Notifikasi Cerdas** — // pengingat jadwal kontrol, // reminder minum vitamin, // dan alert berbasis kondisi kesehatan yang dicatat.

**Integrasi Konten Edukasi** — // artikel mingguan tentang perkembangan janin, // tips nutrisi, dan panduan persiapan persalinan // yang dikurasi khusus untuk konteks pasangan, bukan hanya ibu.

**AI-powered Insight** — // analisis pola dari data yang dicatat // untuk memberikan rekomendasi sederhana // yang personal dan relevan.

**Modul Komunikasi Pasangan** — // ruang berbagi catatan, foto bump, atau pesan singkat // yang terorganisir dalam satu feed kehamilan bersama.

**Integrasi Telehealth** — // fitur untuk menghubungkan pengguna // dengan bidan atau dokter kandungan // langsung dari dalam aplikasi.

## ✦ BAGIAN 5 — KESIMPULAN DAN PENUTUP

---

**[Berdiri di tengah. Nada tenang, kuat, meyakinkan. Ini adalah momen penutup yang berkesan.]**

Baiklah. // Mari kita rangkum apa yang sudah kita bahas hari ini.

**Kita melihat** // bahwa pasar aplikasi kehamilan sedang tumbuh dengan luar biasa pesat — // dengan angka pertumbuhan hampir 20% per tahun. // Ini menunjukkan bahwa kebutuhan akan solusi digital di bidang kesehatan maternal // bukan sesuatu yang bisa diabaikan.

**Kita memahami** // bahwa riset demi riset menunjukkan // betapa pentingnya keterlibatan pasangan // dalam mendukung kesehatan ibu dan perkembangan janin. // Namun ironisnya, // hampir tidak ada platform yang benar-benar memfasilitasi keterlibatan itu // secara aktif dan bermakna.

**Kita melihat** // bahwa kompetitor yang ada // melayani kebutuhan individu dengan baik, // tapi meninggalkan celah besar // di dimensi kolaborasi pasangan.

Kami percaya bahwa masa depan kesehatan maternal // bukan hanya soal fitur yang canggih atau data yang akurat. // Ini soal bagaimana kita **menghadirkan suami** // ke dalam sebuah perjalanan // yang selama ini terlalu sering dijalani sendiri.
